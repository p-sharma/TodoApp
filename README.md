# 📋 Daily Intentional TODO — Android App

> A demo project showcasing an AI-powered product development workflow using **Notion**, **Claude**, and **Jira** — from idea to working code.

---

## 🎯 About This Project

This app is a minimalist Android TODO app built around the philosophy of **intentionality** — users manually re-enter their tasks every day, encouraging active daily reflection rather than mindless carry-over.

But the app itself is secondary. The real purpose of this repository is to demonstrate a modern, AI-assisted product development workflow where Claude acts as the connective tissue between tools and teams.

---

## 🔄 The Workflow

```
Idea → Claude → Notion PRD → Claude → Jira Tickets → Claude Code → Working App
```

### Step 1 — Product Discovery with Claude
The product owner described the app idea in plain English to Claude, which acted as an expert Product Owner to:
- Ask clarifying questions to fill in gaps
- Generate a full **Product Requirements Document (PRD)**
- Iterate on the PRD based on feedback

### Step 2 — PRD Stored in Notion
Claude pushed the finalized PRD directly into **Notion** via its native connector — no copy-pasting. The PRD lives at:
👉 [Daily Intentional TODO — PRD v1.1](https://www.notion.so/34f8e23c89b281b8830ace7ec03b14c3)

The PRD is the single source of truth for the entire team.

### Step 3 — Jira Epics & Stories Generated by Claude
Claude read the PRD from Notion and automatically created **5 epics and 13 stories** in Jira (project key: `DTA`), each with:
- A clear user story ("As a user, I want...")
- Full acceptance criteria
- Correct parent-child epic/story relationships

No manual ticket writing required.

### Step 4 — Sprint Management via Claude
Claude labelled and prioritised Sprint 1 tickets directly in Jira, and tracked progress throughout the sprint — transitioning tickets to **In Progress** and **Done** on the engineer's behalf, just by being told in plain English.

### Step 5 — Code Generation with Claude Code
Engineers picked up Jira stories and used **Claude Code** to implement each feature — pasting the story and acceptance criteria as context, and letting Claude scaffold the implementation.

---

## 🛠 Tools Used

| Tool | Role |
|---|---|
| **Claude (claude.ai)** | Product owner, PRD author, ticket generator, sprint tracker |
| **Notion** | PRD storage and single source of truth |
| **Jira** | Sprint and ticket management |
| **Claude Code** | Feature implementation from Jira stories |

## 💡 Why This Matters

Traditional product development involves a lot of manual, repetitive work — writing tickets, updating statuses, copying requirements into code comments. This demo shows that Claude can automate that entire layer, letting product owners focus on decisions and engineers focus on building.

The workflow is tool-agnostic. Notion and Jira can be swapped for any PRD tool and issue tracker that Claude connects to.
